Course Overview

This project-based course will train students to build a modern, full-stack, web application using JavaScript. Employing skills gained from the first two courses, the student will learn all the facets of constructing a web application, from employing good developer habits and best practices, to building a server supplying a RESTful web service, using build tools and testing code, and finally implementing a single-page application using model/view patterns. Topics include:

  • coding best practices
  • developer work-flow
  • design patterns
  • templating
  • Node.js
  • Express.js
  • Underscore/Lo-dash
  • Backbone.js
  • Grunt.js
  • Mocha, Chai, and Sinon.js

At the conclusion of this course, the student will know how to:

  • Write JavaScript code employing best practices and coding standards, including the use of a style guide and code linting.
  • Understand and use the module pattern to write well-formed, self-contained units of JavaScript code.
  • Use JavaScript to build a server that can serve up pages as well as a RESTful API using Node.js and Express.js.
  • Write JavaScript unit tests using Mocha, Chai, and Sinon.
  • Understand the MV* design pattern as it applies to client-side application development and how to architect a single-page application using Backbone.js.
  • Use tools to facilitate the automation of common tasks such as code linting, test execution, file concatenation and minification.
  • Measure the performance of their code and applications and employ strategies to improve it.

Course Materials:

There is no required text for this course, but mention will be made and references provided for additional relevant texts, online resources, and related materials. These, while not required, provide further background or alternative ways of presenting the same material and topics covered by lecture.

Technical Requirements:

A laptop is required for students who attend in the classroom. If a student attends online, they need to access the class via an Internet connected computer with a fast enough Internet connection to utilize the SyncDL software. The student will also need to have a Github account.
